# Asset Rental Platform

기업 내부에서 사용하는 자산·비품을 효율적으로 관리하고, 사용자들이 손쉽게 대여 신청 및 반납할 수 있도록 만든 **자산·비품 관리 및 대여 시스템**입니다.  
단순한 자산 목록 관리가 아니라, **인증/인가, 대여 워크플로우, 승인 프로세스, 관리자 기능, 배포 환경**까지 고려한 실무형 풀스택 프로젝트를 목표로 개발하고 있습니다.

---

## Overview

이 프로젝트는 사내 자산(노트북, 모니터, 장비, 비품, 시설 등)을 통합 관리하고,  
대여 신청부터 승인, 사용, 반납까지의 과정을 체계적으로 처리할 수 있도록 설계되었습니다.

주요 목표는 다음과 같습니다.

- 기업 내부 자산 및 비품의 통합 관리
- 사용자별 자산 대여 신청 및 반납 처리
- 관리자 승인 기반의 대여 워크플로우 구현
- 역할(Role)에 따른 접근 제어
- Spring Boot + React 기반의 분리형 아키텍처 구성
- AWS 배포를 고려한 서비스 구조 설계

## Main Features

### 1. User & Role Management
- 사용자 정보 관리
- 역할 기반 권한 분리
  - `USER`
  - `MANAGER`
  - `ADMIN`
- 로그인 인증 처리
- 보호된 페이지 접근 제어
- 관리자 전용 페이지 분리

### 2. Asset Management
- 자산 등록 / 수정 / 조회 / 삭제
- 자산 상태 관리
  - 사용 가능
  - 대여 중
  - 비활성화
- 자산 기본 정보 및 카테고리 관리

### 3. Rental Management
- 자산 대여 신청
- 대여 기간 관리
- 대여 상태 변경
  - 신청
  - 승인
  - 반려
  - 대여 중
  - 반납 완료
- 반납 처리 흐름 설계

### 4. Approval Workflow
- 관리자/승인권자의 대여 승인 처리
- 승인 상태 이력 관리
- 단계별 승인 구조 확장 가능하도록 설계

### 5. Admin Features
- 전체 자산 조회
- 대여 신청 목록 관리
- 사용자별 / 자산별 현황 확인
- 관리자 통계 기능 확장 예정

### 6. Frontend UI
- 로그인 페이지
- 자산 목록 / 상세 페이지
- 대여 신청 페이지
- 관리자 전용 페이지
- 권한별 라우팅 처리

## Architecture

### Backend
- REST API 제공
- 인증/인가 처리
- 자산/대여/승인 관련 비즈니스 로직 담당
- 데이터베이스 연동 및 상태 관리

### Frontend
- 사용자 UI/UX 제공
- API 연동
- 권한별 페이지 분기
- 일반 사용자 / 관리자 화면 분리

### Database
- 사용자, 자산, 지점, 영업시간, 예약, 승인 단계 등 핵심 데이터 관리
- 상태값(Enum) 기반 구조 설계

## Expected Benefits

이 시스템을 통해 기업 내부 자산을 보다 체계적으로 관리할 수 있으며,  
대여 신청부터 승인, 사용, 반납까지의 흐름을 표준화하여 운영 효율을 높일 수 있습니다.

또한 역할 기반 접근 제어와 확장 가능한 구조를 통해,  
실제 운영 환경에서도 활용 가능한 자산 관리 플랫폼으로 발전시키는 것을 목표로 합니다.

---

## Project Goal

이 프로젝트는 단순한 CRUD 구현을 넘어,  
실제 서비스 운영을 고려한 **실무형 자산 대여 플랫폼**을 만드는 것을 목표로 합니다.

- 백엔드와 프론트엔드의 역할 분리
- 인증/인가 기반 보안 구조
- 상태 중심의 업무 프로세스 관리
- AWS 배포를 고려한 운영 환경 설계

이를 통해, 개발 역량뿐 아니라  
서비스 구조 설계와 운영 관점까지 함께 담아낸 프로젝트로 완성해 나가고 있습니다.
